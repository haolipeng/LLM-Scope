# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Overview

LLM-Scope (formerly agentsight_go) is a Go rewrite of the original Rust-based AgentSight project. It is an eBPF-based observability framework for monitoring AI agent behavior through SSL/TLS traffic interception and process monitoring. It captures unencrypted request/response data at the kernel level without requiring any code changes to target applications.

The project uses cilium/ebpf with bpf2go for eBPF integration (instead of the original project's libbpf + C userspace approach), Gin for HTTP serving, Cobra + Viper for CLI, and Go's embed.FS for frontend asset embedding.

## Build & Test Commands

```bash
# Full build (BPF code generation + frontend + Go binary)
make build-all

# Individual components
make build-bpf        # Generate BPF Go bindings via bpf2go (requires clang)
make build-frontend   # Build Next.js frontend as static export (frontend/out/)
make build-go         # Compile Go binary only (uses existing .o files)

# Install dependencies (Go modules + bpf2go tool + frontend npm packages)
make deps

# Run tests
make test             # go test -v ./...

# Frontend development
make frontend-dev     # Start Next.js dev server with hot reload

# Clean
make clean
```

## Running

```bash
# Record Claude Code (requires --binary-path for statically-linked BoringSSL)
sudo ./agentsight record -c claude --binary-path ~/.local/share/claude/versions/<version>

# Record Python AI tools
sudo ./agentsight record -c python

# Record with NVM Node.js (statically-linked OpenSSL)
sudo ./agentsight record -c node --binary-path ~/.nvm/versions/node/v20.0.0/bin/node

# Flexible combined monitoring
sudo ./agentsight trace --ssl true --process true --system true --comm claude

# Web UI available at http://127.0.0.1:7395 when using record or trace with --server
```

### Security alerts (Claude Code)

Process-based security rules only see events from the **tracked process tree** (`--comm` matches `/proc/.../comm`, plus descendants). If you monitor **`-c claude`**, running a separate shell script in another terminal **will not** produce valid alerts for that session—invoke equivalent commands **from Claude Code’s terminal / agent** so child processes stay under the tracked tree. See **`scripts/security-alerts-claude-code.md`**. For UI-only DB rows without eBPF, use **`agentsight mock-data`**.

## Architecture

```
eBPF Programs (kernel) → Ring Buffer → Go Collectors → Analyzer Chain → Sinks/Frontend
```

### Key Components

- **`bpf/`** — C eBPF programs. `sslsniff.bpf.c` hooks SSL_read/SSL_write via uprobes; `process.bpf.c` tracks process lifecycle via tracepoints; `stdiocap.bpf.c` captures stdin/stdout/stderr. All use CO-RE and communicate via ring buffers.
- **`internal/bpf/`** — Go bindings generated by bpf2go (`go generate ./internal/bpf/...`).
- **`internal/event/`** — Core `Event` struct and timestamp utilities shared across all packages.
- **`internal/collectors/`** — Collectors (Runners) that load eBPF programs and read events from ring buffers:
  - `base/` — Shared BPF runner infrastructure (ring buffer reading, probe attachment)
  - `ssl/` — SSL/TLS traffic collector
  - `process/` — Process lifecycle collector
  - `system/` — System resource (CPU/memory) collector (reads /proc, no eBPF)
  - `stdio/` — Standard I/O capture collector
- **`internal/pipeline/`** — Streaming framework:
  - `types/` — Core interfaces: `Runner`, `Analyzer`, `Sink`, `MetricsReporter`
  - `transforms/` — Analyzer implementations: SSLFilter, SSEMerger, HTTPParser, HTTPFilter, AuthRemover, ToolCallAggregator, SecurityAnalyzer, ClaudeToolCallAnalyzer
  - `core/` — Analyzer chain builder (`Chain()`)
  - `stream/` — CombinedRunner and MergeStreams for multi-source event merging
  - `sink/` — Output sinks: FileLogger, Output (console), DuckDB persistence
- **`internal/httpserver/`** — Gin-based web server serving frontend assets and `/api/analytics/*` endpoints
- **`cmd/agentsight/`** — CLI entry point with Cobra subcommands
- **`frontend/`** — Next.js/React/TypeScript with Tailwind CSS; `embed.go` uses `//go:embed all:out` to embed static assets

### Data Flow

Events flow through channels (`<-chan *runtimeevent.Event`). Each Runner produces a channel. Analyzers transform channels in sequence via `Chain().Process(ctx, in)`. CombinedRunner merges multiple Runner outputs. MergeStreams merges all streams into one. Sinks consume the final stream for side effects (logging, console output), and HTTP APIs read persisted data.

### Core Interfaces (internal/pipeline/types/types.go)

```go
type Runner interface {
    ID() string
    Name() string
    Run(ctx context.Context) (<-chan *runtimeevent.Event, error)
    Stop() error
}

type Analyzer interface {
    Name() string
    Process(ctx context.Context, in <-chan *runtimeevent.Event) <-chan *runtimeevent.Event
}

type Sink interface {
    Name() string
    Consume(ctx context.Context, in <-chan *runtimeevent.Event)
}

type MetricsReporter interface {
    ReportMetrics()
}
```

## Critical: `--binary-path` and `--comm` Interaction

Applications that statically link SSL (Claude/Bun uses BoringSSL, NVM Node.js uses OpenSSL) require `--binary-path` because there's no system `libssl.so` to hook. When `--binary-path` is specified:

1. sslsniff tries symbol lookup first, then falls back to **BoringSSL byte-pattern detection** for stripped binaries
2. The `--comm` filter is **NOT passed to sslsniff** (only to the process runner) — because `bpf_get_current_comm()` returns the thread name, not the process name. Claude's SSL traffic runs on an "HTTP Client" thread, so `-c claude` would filter out all SSL traffic.

This logic is in `executeTrace()` in `cmd/agentsight/trace.go`.

## Development Patterns

### Adding a New Analyzer

1. Create a new file in `internal/pipeline/transforms/`
2. Implement the `Analyzer` interface (Name + Process methods)
3. The `Process` method takes an input `<-chan *runtimeevent.Event` and returns an output channel
4. Attach via the analyzer chain in the relevant command file

### Adding a New Runner (Collector)

1. Create a new package under `internal/collectors/`
2. Implement the `Runner` interface (ID, Name, Run, Stop methods)
3. `Run()` should return a `<-chan *runtimeevent.Event` channel
4. For eBPF-based collectors, load programs in `Run()` and read from ring buffers
5. Register in the appropriate command file under `cmd/agentsight/`

### Adding a New eBPF Program (bpf2go flow)

1. Create `name.bpf.c` and `name.h` in `bpf/`
2. Add a `//go:generate` directive in the corresponding Go file under `internal/bpf/`
3. Run `make build-bpf` (or `go generate ./internal/bpf/...`) to generate Go bindings
4. Use CO-RE pattern; communicate via ring buffers (not JSON stdout)

### Frontend Development

Set `AGENTSIGHT_FRONTEND_DIR` environment variable to load frontend assets from disk instead of embedded `embed.FS`:
```bash
AGENTSIGHT_FRONTEND_DIR=./frontend/out sudo -E ./agentsight record -c claude
```

## CLI Subcommands

| Command | Description | Key Flags |
|---------|-------------|-----------|
| `record` | Out-of-the-box agent recording with optimized defaults. Enables SSL + process + system + web server. | `--comm` (required), `--binary-path`, `--log-file`, `--server-port` |
| `trace` | Flexible combined monitoring. Toggle each source independently. | `--ssl`, `--process`, `--system`, `--stdio`, `--comm`, `--pid`, `--binary-path` |
| `ssl` | Standalone SSL/TLS traffic monitoring. | `--http-parser`, `--sse-merge`, `--ssl-filter`, `--http-filter`, `--binary-path` |
| `process` | Standalone process lifecycle monitoring. | `--comm`, `--pid`, `--duration`, `--mode` |
| `system` | System resource monitoring (CPU/memory). | `--interval`, `--pid`, `--comm`, `--cpu-threshold`, `--memory-threshold` |
| `stdio` | Standard I/O capture. | `--pid` (required), `--comm`, `--all-fds`, `--max-bytes` |

## Common Issues

- **No SSL capture from Claude/Bun**: Must use `--binary-path` pointing to the actual binary. BoringSSL is statically linked and stripped.
- **`--comm` filter drops all SSL events**: SSL runs on "HTTP Client" thread, not the process name thread. Fixed: `--comm` is auto-skipped for sslsniff when `--binary-path` is set.
- **eBPF permission errors**: Requires `sudo` or `CAP_BPF` + `CAP_SYS_ADMIN`.
- **Port 7395 conflict**: Default web server port. Change with `--server-port`.
- **Frontend not updating**: If using embedded assets, rebuild with `make build-frontend && make build-go`. For development, use `AGENTSIGHT_FRONTEND_DIR`.
